## v4.1.5 - SPA refactor & recovery hardening
- Web UI migrated to external SPA in LittleFS `/www/` (`index.html`, `web.js`, `style.css`).
- Added robust recovery flow:
  - Failsafe embedded setup page when `/www/index.html` is missing.
  - `/setup` route is always available for recovery uploads, even if SPA is broken.
- Config/data loading cleanup in SPA:
  - Settings pages now use `/export_settings` for full configuration objects.
  - Runtime-only status (IP/mode/live state) remains sourced from `/api/status`.
  - Startup now fetches `/api/status` + `/export_settings` in parallel and caches as `ST`/`CFG`.
- Dashboard/Data/Files correctness fixes:
  - CSV export filename adds `_noZero` when "Exclude 0.00L" is active.
  - Live state hint uses timing from `CFG.flowMeter`.
  - Datalog current file is taken from `/api/filelist`; form values from `/export_settings`.
  - Files default tab uses `CFG.hardware.defaultStorageView`.
- Chart.js loader improved: idempotent load, supports local path or CDN with fallback, shows dashboard warning if both fail.
- OTA upload safety checks before flashing: `.bin` extension, minimum size 10 KB, ESP magic byte `0xE9`.
- Version patch increment: 4 → 5.

## v4.1.3 - Wake-up detection accuracy & UI consistency
- Early GPIO snapshot: reads button pins immediately at boot (< 1ms), before loadConfig/initStorage/initHardware
- Captures reed switch state before magnet passes (~50-200ms window), solving misidentification
- Volume-based post-correction: auto-corrects button ID if volume doesn't match expected range
  - PF_BTN with volume ≥ threshold → corrected to FF_BTN
  - FF_BTN with volume ≤ threshold → corrected to PF_BTN
  - Thresholds configurable in Settings / Data Log
  - Enable/disable via checkbox
- Post-correction events logged to btn_log.txt (same storage path as datalog)
  - Format: boot count, GPIO bitmask, early snapshot, correction direction, volume
- CONFIG_VERSION 10: added postCorrectionEnabled, pfToFfThreshold, ffToPfThreshold to DatalogConfig
- UI consistency fixes:
  - Device page-header div properly closed
  - Hardware warning shows full description
  - Save buttons standardized: "Save" for normal, "Save & Restart" for restart-required
  - Form hints use consistent form-hint class
  - Move/Rename popup button renamed to "Apply"
